# 5.1：通知服务（go-server/notify）

## 目标
轻量包，Go 后端其他模块一行代码发 ntfy 通知。

## 产出

### `go-server/notify/notify.go`

```go
package notify

// Send 推送通知到 ntfy
func Send(topic, title, message, clickURL, priority string, tags []string) error
```

- 读取 `NTFY_ADDR` 环境变量，默认 `http://ntfy:80`
- HTTP POST 到 `{NTFY_ADDR}/{topic}`
- Header: Title, Click, Priority, Tags
- Priority: default / low / high / urgent
- 请求超时 5s，失败不 panic

### 常用事件预设（方便其他模块调用）

```go
// InstrumentEmergency 仪器急停
func InstrumentEmergency(instrument, user string) error

// InstrumentRestoreFailed 仪器恢复失败
func InstrumentRestoreFailed(instrument, err string) error

// SecurityAlert 安全告警（登录异常等）
func SecurityAlert(title, detail string) error

// AgentDeadLetter Agent 死信告警
func AgentDeadLetter(taskID, reason string) error
```

## 默认 topic
所有通知发到 `lab-alerts`

## Click URL
`http://10.144.144.12:8000/...`（后续 frp 加上后改）

## 集成点
- `instruments/worker.go`: restore_failed → `NotifyInstrumentRestoreFailed`
- `instruments/handler.go`: emergency-stop → `NotifyInstrumentEmergency`
- `middleware/audit.go`: 安全异常 → `NotifySecurityAlert`
- `py-agent/worker.py`: 死信 → 调 Go API → `NotifyAgentDeadLetter`
