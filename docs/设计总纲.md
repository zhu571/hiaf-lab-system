## 二、核心技术决策（20+ 项）

| # | 决策 | 结论 |
|---|------|------|
| 1 | 后端语言 | Go (chi + 标准库) |
| 2 | Agent 框架 | LightAgent (Python, Apache 2.0) |
| 3 | 数据库 | PostgreSQL 16 |
| 4 | 部署 | Docker Compose @ Rocky Linux 内网物理机 |
| 5 | 前端 | Vue 3 + Element Plus, PWA 双端适配 |
| 6 | ELOG | 逐步退役，数据一次性迁移 |
| 7 | 仪器控制 | E5063A (10.51.12.157:5025) + Hioki (10.51.12.101:3500) |
| 8 | EPICS | PV 归档暂缓，PLC 传感器全量推送 |
| 9 | 传感器推送 | 每分钟 JSON, POST /api/sensors/data, HMAC 鉴权 |
| 10 | 权限 | 模块级 RBAC + 对象级权限 + 项目级隔离 |
| 11 | 交互方式 | PWA + Web（不走 Bot），移动端精简 + 桌面端全功能 |
| 12 | 外网 | frp + VPS (134.175.232.67) + Nginx HTTPS |
| 13 | 认证 | 用户名+密码，首次改密，Argon2id 哈希，JWT + refresh token |
| 14 | 照片 | 本地磁盘，预留 NAS 迁移接口 |
| 15 | LLM Key | Docker 环境变量 |
| 16 | 告警 | ntfy (紧急) + MeoW (日常) |
| 17 | 经验库 | AI 周候选 → 前端页面人工审核 |
| 18 | 过渡期 | 双写 1-2 周 (SQLite + PostgreSQL 并行) |
| 19 | 项目维度 | daily_reports 保留为日容器，daily_report_items 挂项目。生命周期 draft→active→completed→archived |
| 20 | Agent 自动入库 | 三级置信度 + 双门控（Agent候选 + Go规则校验）+ 用户偏好 + 分阶段开启 |
| 21 | AI 问答助手 | MVP 4类问题 + 预定义只读查询函数 + 权限过滤 RAG。统计类问题等结构化字段补全后上 |

## 五、13 个功能模块

| 模块 | 功能 |
|------|------|
| 日志录入 | 多人提交 → AI 三级置信度解析 → 分类入库 |
| 问题管理 | Issue 生命周期（创建/评论/关闭） |
| 经验库 | AI 周候选 → 前端审核 → 入库 |
| 计划管理 | 装配/测试/实验计划，进度跟踪 |
| 项目维度 | 日志/Issue/数据按项目组织，生命周期管理 |
| 仪器控制 | E5063A + Hioki，白名单三级权限 + 互斥锁 |
| 传感器 | PLC 全量推送，鉴权 + 防重放 |
| 实时监控 | 仪表盘：传感器、IOC、备份状态、证书到期 |
| AI 问答 | 只读查询：项目进展/历史问题/经验检索 |
| 用户权限 | 模块级 + 对象级 + 项目级 RBAC |
| 告警通知 | ntfy (紧急) + MeoW (日常) |
| 待审核中心 | Agent 动作 + 经验候选 + 仪器命令集中审批 |
| 运维仪表盘 | 备份状态、frp连接、传感器心跳、证书到期 |

## 六、实施路线图（v6）

| 阶段 | 内容 | 预计 |
|------|------|------|
| **Phase 1** | Go 框架 + 认证/权限/审计 + PostgreSQL + Docker + CI/CD + 迁移脚本 | 2-3 周 |
| **Phase 2** | 日志/问题/经验/项目 API + 前端管理界面 + 运维仪表盘 + 待审核中心 | 3-4 周 |
| **Phase 3** | AI Agent（三级自动入库）+ 问题提取 + 周报生成 + AI 问答 MVP | 2 周 |
| **Phase 4** | 传感器接收 + 仪器控制（只读→写入）+ EPICS Archiver | 2 周 |
| **Phase 5** | frp 穿透 + HTTPS + 告警 + 备份验证 + ELOG 退役 | 1 周 |
